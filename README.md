# Agent — AI-ассистент с веб-поиском на базе LangChain-архитектуры

**Agent** — это fullstack AI-приложение с потоковым чатом, встроенным веб-поиском и сохранением истории диалогов. Проект построен по **клиент-серверной модели** и опирается на **LangChain-архитектуру**: на сервере используется **LangGraph** для оркестрации агентного сценария, вызова инструментов и управления состоянием диалога.

Главная ценность проекта — это не просто чат-интерфейс, а уже собранный **AI-продуктовый каркас**, в котором объединены LLM, поиск в интернете, потоковая генерация ответа, хранение истории и удобный пользовательский интерфейс.

---

## Что уже достигнуто в проекте

Проект уже реализует набор возможностей, который делает его сильной базой для развития в production-сторону:

- **Построен полноценный AI-чат с real-time streaming** — ответы приходят частями через **SSE**, что делает взаимодействие быстрым и живым.
- **Реализован агентный сценарий на основе LangChain/LangGraph** — модель умеет решать, когда отвечать напрямую, а когда подключать инструмент веб-поиска.
- **Интегрирован веб-поиск через Tavily** — агент может находить актуальную информацию и использовать её при генерации ответа.
- **Добавлена прозрачность работы агента** — интерфейс показывает стадии `searching → reading → writing`, а также найденные ссылки.
- **Сделано сохранение истории чатов** — сообщения и диалоги сохраняются в базе данных, а LangGraph использует checkpoint-механику для состояния графа.
- **Реализовано управление диалогами** — можно загружать историю, переименовывать чаты и удалять их.
- **Поддержано разделение пользователей** — на backend уже есть слой привязки диалогов к пользователю через `x-user-id`.
- **Собран современный fullstack-стек** — FastAPI + LangGraph/LangChain на сервере и Next.js + React на клиенте.
- **Подготовлена контейнеризация** — и frontend, и backend запускаются через Docker, что упрощает деплой.
- **Проект уже развернут в облаке** — агент опубликован на **Render** через Docker, доступен онлайн без скачивания и использует **PostgreSQL в Render** для хранения данных.

Итог: проект уже выглядит как **рабочая платформа для AI-ассистента**, а не как локальный эксперимент с одним скриптом.

## Почему этот проект выделяется

### 1. Основа на LangChain-архитектуре

Ключевой архитектурный плюс проекта — использование **LangChain-экосистемы** как базового уровня оркестрации:

- **LangGraph** отвечает за агентный граф и маршрутизацию между моделью и инструментами.
- **LangChain Tools** используются для подключения веб-поиска.
- **LLM с tool calling** позволяет агенту принимать решение о необходимости поиска.
- **Checkpointing** делает архитектуру расширяемой для памяти, восстановления состояния и дальнейшего усложнения логики.

Это означает, что проект уже заложен правильно с точки зрения масштабирования: к текущему графу можно добавлять новые инструменты, память, RAG, авторизацию, аналитику и бизнес-логику без полного переписывания backend.

### 2. Не только генерация текста, а полный агентный цикл

Здесь реализован не просто запрос к модели, а полноценный цикл:

1. пользователь отправляет сообщение;
2. агент анализирует запрос;
3. при необходимости вызывает веб-поиск;
4. получает результаты;
5. формирует финальный ответ;
6. отправляет его в интерфейс в потоковом режиме;
7. сохраняет диалог и сообщения в базе.

### 3. Продуктовый UX уже продуман

Frontend не ограничивается полем ввода. В проекте уже есть:

- список диалогов;
- загрузка истории сообщений;
- новый чат;
- переименование диалога;
- удаление диалога;
- индикаторы процесса поиска;
- состояние генерации ответа и загрузки истории.

Это важный признак зрелости проекта: внимание уделено не только AI-логике, но и пользовательскому опыту.

### Что уже сделано в deployment-части

## Архитектура проекта

```text
Client (Next.js / React)
        │
        │ HTTP + SSE
        ▼
Server (FastAPI)
        │
        ▼
LangChain / LangGraph Agent
   ├── ChatGroq (LLM)
   ├── Tavily Search Tool
   └── LangGraph Checkpointer
        │
        ├── PostgreSQL on Render (checkpoint / state / chat data)
        └── SQLAlchemy models (users / conversations / messages)
```

### Backend

Backend построен на **FastAPI** и организован по модулям:

- `api/` — маршруты и зависимости;
- `core/` — конфигурация и SSE-утилиты;
- `graph/` — сборка LangGraph-графа;
- `services/` — бизнес-логика чатов, пользователей и диалогов;
- `schemas/` — Pydantic-схемы;
- `models.py` / `db.py` — слой данных и ORM.

### Frontend

Frontend создан на **Next.js 15 + React 19** и отвечает за:

- отправку сообщений;
- получение потокового ответа;
- визуализацию сообщений;
- отображение поисковых стадий;
- работу с историей диалогов;
- базовые действия управления чатами.

## Как это работает

1. Пользователь отправляет сообщение из интерфейса.
2. Frontend отправляет запрос на backend.
3. FastAPI передаёт сообщение в агентный граф.
4. **LangChain/LangGraph-архитектура** определяет следующий шаг:
   - ответить сразу;
   - либо вызвать инструмент поиска.
5. При необходимости агент использует **Tavily Search**.
6. Ответ модели стримится обратно в интерфейс через **SSE**.
7. Сообщения и диалог сохраняются в БД.
8. Пользователь видит не только ответ, но и стадии работы агента.

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/volodyapushinskiy/Agent.git
cd Agent
```

### 2. Настройка backend

```bash
cd Server
python -m venv venv
```

**Windows:**

```bash
venv\Scripts\activate
```

**macOS / Linux:**

```bash
source venv/bin/activate
```

Установка зависимостей:

```bash
pip install -r requirements.txt
```

Создай файл `.env` в папке `Server`:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/agent_db
GROQ_API_KEY=your_groq_api_key
TAVILY_API_KEY=your_tavily_api_key
```

Инициализация таблиц:

```bash
python init_db.py
python setup_checkpointer.py
python migrate_add_users.py
```

Запуск backend:

```bash
uvicorn app:app --reload
```

### 3. Настройка frontend

```bash
cd ../Client
npm install
```

Создай `.env.local`:

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
```

Запуск frontend:

```bash
npm run dev
```

Открой в браузере:

```text
http://localhost:3000
```
